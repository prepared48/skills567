**课程时长**：约 60-90 分钟
**目标受众**：AI 小白、对自动化工作流感兴趣的程序员/运营人员
**课程目标**：让没有任何 Agent 开发经验的同学，通过本课程，能从零开始搭建一个具备“自动选题、自动写文、自动配图、自动发布”能力的 AI 助手。

## 详细章节内容

### 0. 开场白与案例演示 (5分钟)
*   **痛点引入**：大家有没有这种经历？想更公众号，但从选题、写文、找图到排版，一套下来至少两三个小时，累得不想动。
*   **效果展示**：今天我们要做的这个 Skill，能让你只说一句话：“写一篇关于‘AI取代程序员’的文章”，5分钟后，文章就已经躺在你的公众号草稿箱里了。（展示最终效果图或视频）
*   **工具准备**：介绍 Trae IDE（或者其他 Agent 开发环境）、Python 环境、微信公众号后台。

### 1. 第一章：什么是 Skill？为什么要用 Skill？ (10分钟)
*   **概念类比**：
    *   如果大模型（LLM）是一个博学的大脑，那么 Skill 就是我们给它装上的“手”和“脚”。
    *   没有 Skill，AI 只能陪你聊天；有了 Skill，AI 才能帮你干活（写文件、调接口、发邮件）。
*   **本项目架构图**：
    *   用户指令 -> **AI 大脑 (解析意图)** -> **Python 脚本 (执行动作)** -> 外部世界 (GitHub 图床、微信后台)。

### 2. 第二章：核心设计 —— 定义 AI 的“大脑” (SKILL.md) (15分钟)
*   **SKILL.md 的作用**：它是给 AI 看的“岗位说明书”。
*   **实战编写**：
    *   **元数据**：定义名字 `wechat-article-generator`。
    *   **Prompt 工程 (重中之重)**：
        *   **人设定义**：你不是冷冰冰的机器，你是“伍六七”，说话要口语化，要有“网感”。
        *   **爆款公式**：标题要扎心，开头要设悬念，结尾要金句。
        *   **结构化输出**：强制 AI 输出 JSON 格式，方便后续脚本处理（比如分开提取标题、正文、图片提示词）。
*   **演示**：在 IDE 里写好 Prompt，测试一下 AI 生成的内容质量。

### 3. 第三章：打造“双手” —— Python 脚本实战 (30分钟)
*   **原理**：AI 生成了内容，但没法直接发到微信，需要 Python 来“搬运”。
*   **脚本拆解**：
    *   **积木A：`generate_images.py` (AI 绘画)**
        *   读取 AI 生成的 Prompt（例如“程序员在加班，赛博朋克风格”）。
        *   调用阿里云 Qwen API，把文字变成图片，保存到本地。
    *   **积木B：`upload_images.py` (图床搬运)**
        *   微信不认本地图片，必须上传到网络。
        *   演示如何用 GitHub API 把本地图片传上去，换回一个 HTTP 链接。
    *   **积木C：`inject_images.py` (内容组装)**
        *   把文章里的 `[图片1]` 占位符，替换成刚才生成的 HTTP 链接。

### 4. 第四章：点睛之笔 —— 自动配图与发布 (15分钟)
*   **为什么需要这一步**：写好了 markdown，怎么变成微信公众号的草稿？
*   **`publish_article.py` 实战**：
    *   介绍微信公众号 API（或者使用第三方简化接口，如 LimyAI）。
    *   演示如何把最终的 Markdown 内容，一键推送到草稿箱。
*   **关键配置**：
    *   `.env` 文件的重要性：不要把密码写在代码里！(API Key, AppID 等)。

### 5. 第五章：完整运行与避坑指南 (10分钟)
*   **全流程演示**：从输入指令到公众号后台看到草稿的完整过程。
*   **常见坑**：
    *   **图片裂了**：讲讲 GitHub 防盗链和 CDN 加速。
    *   **依赖报错**：`pip install requests` 别忘了。
    *   **AI 也就是个复读机**：Prompt 没写好，文章就像八股文，强调 Prompt 迭代的重要性。

### 6. 互动答疑
*   回答弹幕问题。
*   预告下期内容（例如：如何接入热点新闻 API 实现自动追热点）。
