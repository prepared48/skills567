# 从零打造全能公众号助手：Skill 开发三步进化论
## —— 给 AI 小白的第一份 Agent 开发实战手册

**课程目标**：通过“三步走”的进化策略，带领学员从零开始，逐步搭建并升级一个能自动写文、配图、发布的 AI 助手。
**核心理念**：小步快跑，每一步都有正反馈。

---

## 🚀 导读：为什么我们要用“进化论”来学习？
*   **传统误区**：一上来就写一堆 Python 代码，环境还没配好，人先劝退了。
*   **我们的玩法**：
    *   **V1.0 纯享版**：5 分钟搞定，让 AI 先学会“写文章”。（只要会打字就能做）
    *   **V2.0 视觉版**：给 AI 装上“眼睛”和“画笔”，让文章图文并茂。（开始接触一点点代码）
    *   **V3.0 躺平版**：打通最后一公里，一键发布到后台。（成为真正的 Agent 开发者）

---

## 📝 第一阶段：V1.0 纯享版 —— 打造最强“文字秘书”
**目标**：输入一个主题，AI 自动写出一篇结构完美、金句频出的 Markdown 文章。
**耗时**：约 10-15 分钟
**技能点**：Prompt 工程、SKILL.md 基础

### 1.1 什么是 Skill？（简单理解）
*   把 Skill 想象成给 AI 发的“岗位说明书”。
*   没有 Skill，AI 是个只会陪聊的网友；有了 Skill，AI 就是你的专属员工。

### 1.2 实战：编写你的第一份“岗位说明书” (SKILL.md)
*   **任务定义**：告诉 AI “你是谁”（公众号爆款写手）。
*   **Prompt 魔法（核心）**：
    *   **人设注入**：不要机械音，要“网感”，要“口语化”。
    *   **结构模板**：标题 -> 痛点 -> 观点 -> 金句 -> 结尾。
    *   **避坑指南**：禁止爹味说教，禁止正确的废话。
*   **动手环节**：在 IDE 里创建 `wechat-article-generator/SKILL.md`，复制模板，填入 Prompt。

### 1.3 见证奇迹：第一次运行
*   **指令**：“写一篇关于‘程序员35岁危机’的文章”。
*   **结果**：看到 AI 吐出一篇排版精美、内容扎实的文章。
*   **复盘**：这就是 V1.0，你已经拥有了一个 24 小时待命的写作助手。

---

## 🎨 第二阶段：V2.0 视觉版 —— 升级为“图文主编”
**目标**：文章不仅要有字，还要有 AI 自动生成的配图，且图片能稳定显示在微信里。
**耗时**：约 30-40 分钟
**技能点**：Python 脚本调用、API 基础、图床原理

### 2.1 痛点：光有文字不够看
*   纯文字没人看，找图又怕侵权。
*   微信公众号不支持本地图片，直接复制粘贴会“裂图”。

### 2.2 进化：给 AI 装上“画笔” (generate_images.py)
*   **原理**：让 AI 在写文章时，顺便把“画面感”描述出来（Prompt），然后找个画师（阿里云 Qwen）画出来。
*   **操作**：
    *   配置阿里云 API Key（几十块钱能用很久）。
    *   添加一段 Python 脚本，自动读取 Prompt 并下载图片。

### 2.3 进化：给 AI 装上“搬运工” (upload_images.py)
*   **原理**：本地图片发不出去 -> 传到云端（GitHub） -> 拿到链接 -> 塞回文章里。
*   **操作**：
    *   注册 GitHub 账号，创建一个公开仓库。
    *   配置 Token。
    *   运行脚本，把本地图片变成 `https://...` 的链接。

### 2.4 成果验收
*   **指令**：“写一篇关于‘未来城市’的文章，要配图”。
*   **结果**：文章里不仅有字，还插入了 AI 画的赛博朋克风格插图。

---

## 📨 第三阶段：V3.0 躺平版 —— 成为“全栈运营”
**目标**：连复制粘贴都省了，直接推送到公众号草稿箱，手机上预览发步。
**耗时**：约 20 分钟
**技能点**：API 接口调用、环境变量管理 (.env)

### 3.1 痛点：最后一步最烦人
*   虽然文章好了，图也有了，但还要登录微信后台、扫码、复制、调整格式……这一步最消磨热情。

### 3.2 终极进化：打通“最后一公里” (publish_article.py)
*   **原理**：微信开放了接口，我们可以用代码“伪装”成浏览器，把文章“快递”进去。
*   **操作**：
    *   获取微信公众号的 AppID 和 AppSecret。
    *   （可选）使用中间件（如 LimyAI）简化开发，避免复杂的 Token 刷新逻辑。
    *   编写发布脚本。

### 3.3 安全第一：保护你的秘密 (.env)
*   **关键知识**：千万别把密码写在代码里！
*   **操作**：创建 `.env` 文件，把 Key 都藏在这里，并告诉 Git “不要上传它” (`.gitignore`)。

### 3.4 最终验收：High Moment
*   **指令**：“写一篇‘猫咪的职场生存法则’，配萌宠图，直接发布”。
*   **结果**：
    1.  IDE 疯狂输出日志（生成中... 画图中... 上传中... 发布中...）。
    2.  手机微信“叮”一声，草稿箱收到新文章。
    3.  **这一刻，你完成了一个全栈 AI Agent 的开发！**

## 🎁 结语与彩蛋
*   **总结**：我们通过 V1 -> V2 -> V3，像搭积木一样，把一个简单的想法变成了一个强大的工具。
*   **扩展思路**：
    *   想追热点？接入新闻 API。
